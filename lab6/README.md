1. Загальний опис проекту
Метою роботи є розробка модульного застосунку на C++ для захоплення, обробки та відображення відеопотоку в реальному часі. Програма реалізує перемикання між різними режимами фільтрації та інтерактивну взаємодію з користувачем через маніпулятор "миша".

2. Архітектура та структура коду
Проект побудований за принципами об'єктно-орієнтованого програмування (ООП). Весь функціонал розділений на логічні модулі:
   CameraProvider.hpp: Модуль захоплення відео. Використовує клас cv::VideoCapture. Реалізовано логіку автоматичного зациклення (loop) відеофайлу, щоб при завершенні ролика програма не закривалася, а починала відтворення спочатку.
   FrameProcessor.hpp: Модуль обробки зображення. Містить методи для накладання фільтрів та малювання графічних примітивів (кіл) за координатами миші.
   KeyProcessor.hpp: Модуль керування. Обробляє коди натиснутих клавіш (ASCII) та перемикає стани програми (Enum Mode).
   main.cpp: Точка входу. Об'єднує всі модулі в єдиний цикл обробки, ініціалізує вікно OpenCV та встановлює Callback-функцію для миші.

3. Використані методи обробки (OpenCV)
У програмі реалізовано 4 основні режими:
   Original: Відображення без змін.
   Canny Edge Detection: Використання алгоритму cv:Canny для виділення контурів об'єктів.
   Gaussian Blur: Математичне розмиття за допомогою ядра cv::GaussianBlur (розмір ядра 15x15).
   Inversion: Попіксельна інверсія кольорів через операцію cv::bitwise_not.

4. Чому було обрано відеофайл замість веб-камери?
Під час тестування у віртуальному середовищі VMware Workstation було виявлено технічну несумісність драйверів USB-контролера з форматом стиснення веб-камери (MJPEG/YUY2). Це призводило до появи артефактів (суцільний зелений екран).
Рішення: Для забезпечення стабільної демонстрації алгоритмів обробки було прийнято рішення використовувати відеофайл video.mp4. Це дозволяє:
-Гарантувати високу якість вхідного сигналу для фільтрів.
-Показати роботу програми незалежно від наявності специфічного обладнання.
-Продемонструвати гнучкість класу CameraProvider, який підтримує як фізичні пристрої, так і файлові потоки.

5. Інструкція із запуску (Автоматизація)
Для спрощення перевірки створено три Bash-скрипти:
   preinstall.sh: Автоматично встановлює cmake, g++ та бібліотеку libopencv-dev. Використовує менеджер пакетів apt.
   build.sh: Реалізує процес збірки "out-of-source". Створює папку build, запускає конфігурацію CMake та компіляцію через make.
   run.sh: Запускає скомпільований бінарний файл lab6_app.

6. Детальний опис функціональних модулів (Технічна частина)
  Механізм Callback-функції для миші:
Для малювання було використано статичний метод cv::setMouseCallback. Оскільки С++ вимагає передачі вказівника на функцію, а ми працюємо в ООП-стилі, було використано передачу вказівника на об'єкт FrameProcessor через параметр void* userdata. Це дозволило змінювати стан об'єкта класу безпосередньо з глобальної функції обробки подій.
  Синхронізація кадрів:
Затримка cv::waitKey(30) вибрана не випадково. Вона забезпечує приблизно 33 FPS, що відповідає стандарту плавного відео та дає процесору час на обробку фільтрів між кадрами.

7. Використані системні команди та інструменти
  CMake (версія 3.10+): Використовується для автоматизації збірки. Команда find_package(OpenCV REQUIRED) дозволяє програмі бути переносимою між різними дистрибутивами Linux (Ubuntu, Debian, Fedora), автоматично знаходячи шляхи до заголовних файлів (/usr/include/opencv4) та бібліотек.
  G++ та стандарт C++11: Проект використовує стандарт C++11 (або вище) для підтримки переліків enum class та покращеного керування пам'яттю.
  Bash-автоматизація: Скрипти написані з використанням стандартної оболонки /bin/bash для забезпечення швидкого розгортання проекту на "чистій" системі однією командою.

8. Опис математичних алгоритмів (Теоретична довідка)
Алгоритм Кенні (Canny): Використовує багаторівневий підхід для виявлення широкого спектру країв. Включає розмиття по Гаусу (для шуму), пошук градієнтів та відсікання слабких пікселів.
Гаусове розмиття: Операція згортки зображення з ядром Гауса (матриця коефіцієнтів). Це математично зменшує високочастотні деталі зображення.

Команди для запуску:
  chmod +x *.sh
  ./preinstall.sh
  ./build.sh
  ./run.sh
